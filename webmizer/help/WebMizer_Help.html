Can't run WebMizer

- WebMizer is a set of tools for Debian GNU/Linux (and derivatives like Ubuntu).
- Make sure that you have FFmpeg 0.6 or above installed.
- Check the permissions tab of the *.sh files, "Allow executing file as program" should be marked.


Need more info or support about FFmpeg

- "FFmpeg is a complete, cross-platform solution to record, convert and stream audio and video." [http://ffmpeg.org]
- We should also add that FFmpeg is an awesome tool :)
- WebMizer can be seens as an interface layer for using FFmpeg to create WebM videos.
- Because "FFmpeg 0.6 contains WebM container support and substantial VP8 support" [http://www.webmproject.org/tools/], to use WebMizer you should make sure that you have FFmpeg 0.6 or above installed.

Need more info on video quality

Basic speed/quality control parameters:
Best Best quality output but is slow.
Good Probably what most users use most of the time. Will give quality that is usually very close to and even sometimes better than that obtained with Best but the encoder will typically run faster. 
Fastest Significant boost to encode speed but will start to have a more noticeable impact on quality and may also start to effect the accuracy of the data rate control.

For more info on this issue check:
http://www.webmproject.org/docs/encoder-parameters/
http://www.webmproject.org/tools/
http://ffmpeg.org/ffmpeg.html


Need more info on audio quality

- WebMizer uses Ogg Vorbis for audio encoding.
- You can choose from a 0-10 range for audio quality. 0 means less quality and 10 means best quality.
- If you want more specs about the audio quality, please check the table below. This table is a copy/paste from info provided at the Hydrogenaudio Knowledgebase.

Switch VBR target
(kbit/s) 		VBR range
(kbit/s) 		Channel
Coupling 		Noise
Normalization 		Note
-q -2 ~32 ~32 – ~64 point/lossless yes (1)
-q -1 ~48 ~48 – ~64 point/lossless yes (2)
-q 0 ~64 ~64–~80 point/lossless yes
-q 1 ~80 ~80 – ~96 point/lossless yes (3)
-q 2 ~96 ~96 – ~112 point/lossless yes
-q 3 ~112 ~112 – ~128 point/lossless yes

-q 5 ~160 ~160 – ~192 point/lossless no (4)

-q 8 ~256 ~256 – ~320 lossless no
-q 9 ~320 ~320 – ~500 lossless no
-q 10 ~500 ~500 – ~1000 lossless no


(1) Only supported on aoTuVb3 and newer
(2) Bitrate of 48 kbit/s is only for aoTuVb3 and newer. Earlier versions and Xiph.org versions use a bitrate of 45 kbit/s
(3) Reports seem to indicate that aoTuV Release 1 -q 1 provides good quality for streaming
(4) Most users agree -q 5 achieves transparency, if the source is the original or lossless. It is not transparent in the case of transcoding from lossy source (strongly frowned upon)

If you need to know what lowpass settings are used for each quality level, see this HA thread. (It is not recommended that you adjust these)
